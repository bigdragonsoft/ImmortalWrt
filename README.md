# 📖 固件使用指南

在使用本固件之前，请务必仔细阅读以下说明，以避免配置错误导致无法连接。

## ⚠️ 旁路由用户必读 (单网口设备)

> **请勿在编译配置中直接修改默认 IP 地址试图设置为旁路 IP，这会导致网络配置混乱。**

1.  **默认模式**：单网口设备默认采用 **DHCP 客户端模式**（类似 NAS），它会自动从上级路由器获取 IP 地址。
2.  **如何访问**：
    *   将设备连接到上级路由器。
    *   进入上级路由器的后台，查看分配给 ImmortalWrt 的 IP 地址。
    *   通过该 IP 地址访问后台页面。
3.  **配置旁路 IP**：
    *   进入 ImmortalWrt 后台后，再根据主路由的网段，**手动配置** 静态 IP 地址作为旁路由 IP。

---

## 🌐 正常路由模式必读 (多网口设备)
适用于拥有 2 个或以上网口的设备。

*   **WAN 口**：默认自动获取 IP (DHCP) 或拨号。
*   **LAN 口**：默认 IP 为 `192.168.100.1`，并开启 DHCP 服务给子设备分配 IP。
*   **修改默认 IP**：
    *   如果默认 IP `192.168.100.1` 与光猫或其他路由器冲突，可以在编译配置中修改（例如改为 `192.168.80.1`）。
    *   大多数情况下，**无需更改**。

---

## ⚙️ 固件默认属性速查

| 设备类型 | 默认网络模式 | 说明 |
| :--- | :--- | :--- |
| **单网口设备** | **DHCP 客户端** | 自动获取 IP。请先连接上级陆由器，查看后台分配的 IP 后访问。 |
| **多网口设备** | **WAN: DHCP / LAN: 静态** | LAN 口默认 IP: `192.168.100.1`。<br>默认 `eth0` 为 WAN，其余为 LAN。 |

**其他说明：**
*   **PPPoE 拨号**：若在 Workflow 中勾选了拨号信息，WAN 口将配置为 PPPoE 模式。建议拨号用户在使用前重启一次光猫。
*   **自定义配置**：以上默认行为均可通过 `files/etc/uci-defaults/99-custom.sh` 脚本进行调整。

---

## 🔒 特别说明：WAN 口防火墙

出于易用性考虑，本固件默认 **开启** 了 WAN 口的入站防火墙规则（允许入站），方便初次通过 WAN 口访问管理。

**安全建议**：
初次调试完毕，确认网络正常后，建议 **关闭** WAN 口入站许可。

**操作路径**：
`网络` -> `防火墙` -> `WAN 口` -> `入站数据` -> 选择 `拒绝` -> `保存并应用`
